# PLAN.md — repo-hand-off

## Context

VISION.md で定義された「開発の状態を保存・共有し、人やAI間でシームレスに引き継ぐCLIツール」の実装計画。
Go + cobra で CLI を構築。MVP は export（HANDOFF.md 生成）と prompt（AI向けコンテキスト出力）に集中する。
非Git環境でも壊れず、取得できる情報だけで動作する。

## Phase 2: `handoff export` コマンド

`handoff export` で HANDOFF.md を生成する。非Git環境では Git セクションをスキップする。

- [ ] Git 状態収集（ブランチ、最新コミット、未コミット変更、リモートURL）— Git なしなら graceful skip
- [ ] プロジェクトファイル検出・読み取り（VISION.md, PLAN.md, LESSONS.md, README.md, CLAUDE.md）
- [ ] 簡易ディレクトリ構造（浅い階層、.gitignore 準拠除外）
- [ ] HANDOFF.md 生成・書き出し
- [ ] `--output` フラグ（出力先指定、デフォルト: `./HANDOFF.md`）

### HANDOFF.md フォーマット

```markdown
# HANDOFF.md
> Generated by repo-hand-off at {timestamp}

## Project
- Repository: {remote_url or "N/A"}
- Branch: {branch} @ {short_hash}
- Uncommitted changes: {yes/no}

## Vision
{VISION.md 内容 or "Not found"}

## Plan
{PLAN.md 内容 or "Not found"}

## Lessons
{LESSONS.md 内容 or "Not found"}

## Current State
{recent commits summary}

## Directory Structure
{tree output}
```
